# AGENTS.md — 프로젝트 규칙 (단일 소스)

이 문서는 모든 AI 코딩 에이전트가 따라야 할 규칙의 **단일 소스**다.
다른 컨텍스트 파일(`CLAUDE.md` 등)은 이 파일을 가리키는 한 줄 포인터만 둔다.
규칙이 새로 생기면 prose가 아닌 **도구**(pre-commit, CI, 테스트)에 박는 것을 우선하고,
도구화가 어려운 경우에만 본 문서에 추가한다.

상세 설계와 단계 정의는 [PLAN.md](./PLAN.md)에 있다. 본 문서는 행동 규칙만 둔다.

---

## 1. 작업 방식
- 단계별로 진행한다. 각 단계 끝에서 실제로 실행해 동작을 검증한 뒤 다음으로 넘어간다.
- **검증 전 임의 커밋 금지.** 사용자 OK 후에만 커밋한다.
- **한 커밋 = 한 논리 변경.** Conventional Commits (`feat:`, `fix:`, `chore:`, `docs:`, `ci:`, ...).
- 코드 변경은 `PLAN.md`의 단계 정의를 따른다. 단계 외 작업이 필요하면 먼저 `PLAN.md`를 갱신하고 확인받는다.

## 2. 보안 (강제)
- 시크릿(`ANTHROPIC_API_KEY`, `SLACK_WEBHOOK_URL`, `SMTP_*` 등) **하드코딩 금지.**
  환경변수 + GitHub Secrets로만 읽는다. 저장소엔 `.env.example`만 둔다.
- pre-commit `gitleaks` 훅으로 커밋 단계에서 차단한다. 동일 검사를 CI에서 한 번 더 돌린다.
- 외부에서 긁어온 텍스트의 개인정보(이름·이메일·전화)는 요약·로그에 노출하지 않는다.
  AI 프롬프트에 명시 + 발송 직전 정규식 1차 차단을 함께 둔다.

## 3. 코드 규칙
- 외부 호출(HTTP, API)은 인터페이스로 추상화하여 단위 테스트에서 모킹 가능하게 분리한다.
  실 호출은 별도 통합 단계에서만 검증한다.
- 모든 외부 호출에 **타임아웃과 실패 로그**를 둔다. 한 소스의 실패가 전체 파이프라인을 멈추지 않도록 격리한다.
- 의존성은 최소. 새 의존성 추가 시 `PLAN.md`에 이유를 적는다.
- 다이제스트 출력은 한국어로 작성한다.

## 4. Anthropic Claude API
- 구조화 출력은 **tool-use 강제** 방식으로 한다 (`tool_choice={"type":"tool","name":"..."}`).
  Anthropic API에는 OpenAI 류의 `response_format`이 없으므로 시도하지 않는다.
- 기본 모델은 `claude-haiku-4-5-20251001`. 품질 부족 시 `claude-sonnet-4-6`으로 승격.
- 시스템 프롬프트·정책은 prompt caching 블록으로 묶는다.

## 5. 컨텍스트 파일 정책
- 본 파일이 단일 소스. **규칙을 다른 파일에 중복 작성하지 않는다.**
- 사용 중인 에이전트 툴이 자기 컨텍스트 파일을 따로 읽는 경우에만 한 줄 포인터를 둔다.
  현재 등록된 포인터:
  - Claude Code → `CLAUDE.md`에 `@AGENTS.md`
- 다른 툴(Cursor의 `.cursorrules` 등)은 사용 시점에 추가한다. 기본은 만들지 않는다.

## 6. 실수 → 규칙 승격
- 같은 실수를 반복하지 않도록, 교정된 교훈은 즉시 본 문서에 규칙으로 추가한다.
- 가능한 한 도구(훅·CI·테스트)에 박아 자동 강제한다. prose 규칙은 최후의 수단.
